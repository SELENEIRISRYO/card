> 《软件工程》课程实验作品 · 2026 春季学期

---

## 作品名称

**我的个人数字名片页** — My Digital Business Card

---

## 作品简介

这是一个使用纯 HTML + CSS 构建的个人数字名片页面，旨在以网页的形式展示个人基本信息、自我介绍、技能与兴趣，并通过 Gitee Pages 公开发布，形成一个可供他人访问的个人作品页面。

本页面没有使用任何前端框架或 JavaScript，仅依赖 HTML 语义结构与 CSS 实现完整的视觉效果与交互动画，体现了"用最少的依赖做最完整的事"的工程思维。

---

## 页面主要内容说明

| 区域 | 内容 |
|------|------|
| Hero 区域 | 头像、姓名（中英文）、学号、专业、个性签名、联系方式 |
| 关于我 | 不少于 100 字的自我介绍 |
| 兴趣爱好 | 5 个兴趣标签，渐变胶囊样式 |
| 技能栈 | 5 项技能 + 动画进度条 |
| 外部链接 | Gitee / 个人博客 / GitHub 共 3 个外部链接卡片 |
| Footer | 版权信息与课程说明 |

## 页面公开访问方式

### 方式一：Gitee Pages（推荐）

1. 在 Gitee 上创建仓库（如：`digital-card`）
2. 将本项目所有文件推送到仓库 `main` 分支
3. 进入仓库 → **服务** → **Gitee Pages**
4. 选择分支 `main`，根目录 `/`，点击启动
5. 稍等片刻后获得公开访问链接：
   ```
   https://your-username.gitee.io/digital-card/
   ```

### 方式二：本地预览

直接用浏览器打开 `index.html` 文件即可预览。

## 本作业中最能体现"软件作品"而非"单个页面"的一点

> **CSS 自定义属性（变量）的系统化设计**。

本项目在 `style.css` 的 `:root` 中集中定义了颜色、间距、过渡、阴影等设计 token，整个页面的视觉风格由这一组变量统一驱动。这意味着：

- 若要更换主题色，只需修改 2 行变量，全页面同步更新；
- 卡片、按钮、进度条的样式遵循统一规范，具备可维护性；
- 新增区块时，直接复用现有 `.card` 类即可保持一致性。

这种"设计系统思维"是软件工程中**可维护性与可扩展性**原则在前端的具体体现，而不只是"把内容写进一个文件"的页面堆砌。

## AI 辅助开发说明

### 是否使用 AI

是，使用了 AI 编程助手辅助本项目的开发。

### 使用了哪些工具

- **WorkBuddy（AI 编程助手）**：用于生成 HTML 结构与 CSS 样式的初始版本
- **浏览器开发者工具**：用于调试样式和验证布局

### 关键提示词（2—4 条）

1. `"帮我做一个个人数字名片页，清新渐变风格，包含头像、姓名、学号、自我介绍、兴趣标签、技能进度条和外部链接，纯 HTML+CSS，不用 JavaScript"`
2. `"CSS 进度条用 animation + CSS 变量 --pct 实现，不用 JS"`
3. `"Hero 区域加半透明装饰圆形背景，用 backdrop-filter 毛玻璃效果做卡片"`
4. `"生成一个渐变色卡通占位头像 SVG，带微笑表情，提示用户替换为本人照片"`

### AI 主要帮助了哪些部分

- **HTML 语义结构**：AI 生成了完整的 section/nav/main/footer 结构，语义正确
- **CSS 渐变与动画**：进度条动画、Hero 背景形状、卡片 hover 效果由 AI 给出初稿
- **响应式布局**：媒体查询的断点设置由 AI 建议，经验证后采用
- **SVG 头像生成**：AI 直接生成了符合要求的占位卡通头像

### 我对 AI 结果做了哪些修改

- 调整了色彩变量，把默认蓝色系改为更有个性的紫粉渐变
- 将 AI 生成的英文注释改为中文，方便自己后续维护
- 修改了自我介绍的内容，替换为真实（或接近真实）的个人信息
- 删除了 AI 生成的冗余 `<div>` 嵌套，简化了 DOM 结构
- 调整了字体大小和行高，使阅读体验更舒适

### 我认为 AI 最不可靠的一点是

> **AI 无法替你做"个人化"的决定。**

AI 生成的内容是合理但通用的——它不知道你真正的兴趣是什么、你的技能实际到什么水平、你想给别人留下什么印象。如果不加修改地直接使用，名片页会变成一个"看起来像你但其实不是你"的空壳。  
AI 擅长解决结构和技术问题，但内容的真实性与个性化，必须由人来完成。
